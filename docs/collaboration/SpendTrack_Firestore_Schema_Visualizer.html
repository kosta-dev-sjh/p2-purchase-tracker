SpendTrack • Firestore Model

지금 코드 기준으로 바로 구현 가능한 Firestore 구조

현재 앱은 사실상 프로필, 거래, 카테고리 3개 엔티티로 움직입니다. 그래서 1차 연동도 그 3축에 맞춰 최소 구조로 설계하는 편이 가장 안전합니다. 인증은 Firebase Auth, 데이터는 Firestore, 아바타 이미지는 Storage로 분리하는 구성이 현재 코드와 가장 잘 맞습니다.

3 1차 구현 핵심 컬렉션: `users`, `transactions`, `categories`
4 즉시 필요한 거래 액션: 추가, 수정, 삭제, OCR 병합
2 권장 복합 인덱스: 월별 조회, OCR 매칭 후보 조회

Collection Flow

users/{uid}

User Profile

  • displayName, nickname, email
  • avatarUrl, onboardingSeen
  • createdAt, updatedAt
→
users/{uid}/transactions/{txId}

Transactions

  • 월별 조회, 검색, 수정, 삭제
  • CSV 벌크 추가
  • OCR 결과 병합
→
users/{uid}/categories/{categoryId}

Categories

  • 표준 5종 seed
  • 색상/이름 수정
  • 사용자 정의 카테고리 추가

User Document

Field Type Note
displayName string 현재 `profile.name` 대응
nickname string Settings 및 헤더 표시용
email string Auth 값 캐시
avatarUrl string | null Storage URL 권장
passwordChangedAt string 표시용 필드만 유지
onboardingSeen boolean localStorage 플래그 대체 가능
비밀번호 저장 금지 Auth와 1:1 대응 AppShell / Settings 공용

Category Document

Field Type Note
name string 화면 라벨
color string hex color
isStandard boolean 표준 카테고리 여부
isLocked boolean `etc` 보호용
sortOrder number 표시 순서
표준 ID: living/fashion/digital/food/etc 초기 seed 필요

Transaction Document

Field Type Why it exists
type "expense" | "income" 현재 UI 핵심 구분
dateText string 기존 화면과 포맷 호환
dateTs Timestamp 정렬 및 범위 조회
monthKey string 월 단위 목록 조회
platform enum OCR 매칭, 필터링
categories string[] 현재 다중 카테고리 구조 유지
title string 표와 검색의 기본 키
amount number 지출 음수, 수입 양수 규칙 유지
absAmount number OCR 매칭 후보 조회용
status enum purchase/cancel/refund/sub/etc
detail.items array 상품 목록 저장
detail.source "OCR" | "MANUAL" 상품 생성 출처

Required Queries

Use Case Query Shape
월별 거래 목록 `where(monthKey == x) + orderBy(dateTs desc)`
OCR 매칭 후보 `where(monthKey == x, platform == x, absAmount == x)`
카테고리 목록 `users/{uid}/categories` 전체 조회
monthKey + dateTs 인덱스 monthKey + platform + absAmount 인덱스

Security Rule Principle

match /users/{userId} {
  allow read, write: if request.auth != null
                     && request.auth.uid == userId;

  match /transactions/{transactionId} {
    allow read, write: if request.auth != null
                       && request.auth.uid == userId;
  }

  match /categories/{categoryId} {
    allow read, write: if request.auth != null
                       && request.auth.uid == userId;
  }
}

Recommended Team Split

Transactions CRUD

3

Profile + Categories

4

OCR Attach Flow

OCR 저장, 기존 거래 매칭, `detail.items` 병합 업데이트를 마무리합니다.
