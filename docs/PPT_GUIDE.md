# SpendTrack 문서 및 발표자료 재정비 작업 요청

## 1. 작업 목적

현재 SpendTrack 프로젝트는 1차 배포가 완료된 상태이며,  
현재 코드와 실제 구현 상태를 기준으로 다음 두 가지를 정리해야 한다.

1. 기존 md 문서들을 현재 프로젝트 상태에 맞게 업데이트
2. 최종 발표용 PPT 자료 초안 재구성

※ 이번 발표는 "중간 발표 없이 바로 최종 발표"이므로,  
현재 구현 상태 + 실제 서비스 가능성 + 향후 확장성을 함께 보여주는 방향이어야 한다.

---

# 2. 반드시 기준으로 삼을 것

## 가장 우선 기준
- 현재 코드 구조
- 현재 실제 구현 상태
- 현재 Firebase / OCR / CSV / AI fallback 구조

즉, 과거 문서보다 **현재 코드가 최우선 진실(source of truth)** 이다.

# 4. 문서 수정 작업 (md)

현재 md 파일들을 다음 기준으로 업데이트

## 수정 기준
- 현재 실제 구현된 기능 기준
- 과장 제거
- 미구현 기능은 future plan으로 분리

## 반드시 반영할 핵심 기능
- OCR 이미지 업로드
- CSV/XLSX 거래내역 입력
- AI fallback 구조
- 반복결제 탐지
- 소비 분석 대시보드
- Firebase 기반 데이터 동기화

## 현재 구조 정확히 반영
- tesseract.js 브라우저 1차 OCR
- Gemini fallback
- Firestore onSnapshot 구조
- Firebase Functions 사용 구조

이를 이용해서 ppt 작업도 행하며, 우리가 내부적으로 남아있는 자료도 꾸려지므로 네게 부탁하는 것임.

# 6. 발표 핵심 흐름

다음 흐름으로 재구성

## 1. 왜 SpendTrack인가
필요성 제시

포함:
- 온라인 소비 증가
- 구매 기록은 남지만 관리되지 않음
- 플랫폼별 구매내역 분산
- 충동구매 체감 어려움

→ 관련 근거 조사 후 신뢰 가능한 출처 포함

---

## 2. 타겟 사용자

강사 피드백 반영:

- 온라인 구매 친숙한 20대~40대
- 온라인 쇼핑 비중 높은 사용자
- 반복결제 관리 필요한 사용자

→ 근거 조사 + 출처 포함

---

## 3. 문제 정의

예:
- 어디서 무엇을 샀는지 기억 어려움
- 카드내역과 쇼핑내역 연결 어려움
- 반복결제 놓침

---

## 4. 기존 서비스와 차별점

예:
- 단순 가계부 아님
- 쇼핑 단위 추적
- 플랫폼 단위 OCR 분석
- CSV + 이미지 혼합 입력

---

## 5. 핵심 기능

반드시 포함:

- 이미지 분석
- CSV/XLSX 분석
- AI fallback
- 반복결제 추적
- 소비 인사이트

# 7. 운영비 자료 반드시 포함

현재 코드 기준 운영 구조:

- 정적 프론트

핵심 메시지:
현재 운영비가 매우 낮음

## 월 추정비용
- ₩1,300 ~ ₩2,500 수준

## 강조
비용 대부분 Gemini fallback

## 사업성
- 구독형 확장 가능
- OCR/반복결제/AI 리포트 유료화 가능
